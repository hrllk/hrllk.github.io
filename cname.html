CNAME 레코드: 도메인 별칭 설정의 모든 것

1. CNAME(Canonical Name) 레코드란?

CNAME 레코드는 하나의 도메인 이름을 다른 도메인 이름(별칭)으로 매핑해 주는 DNS 레코드 유형입니다. 특정 도메인이 실제 IP 주소가 아닌, 다른 도메인을 가리키도록 할 때 사용합니다.

2. 사용 예시

예를 들어, www.myblog.com 접속 시 실제로는 myblog.com 페이지를 보여주고 싶을 때 다음과 같이 설정할 수 있습니다.

# DNS 설정 예시
example.com     A       192.168.1.1
sub.example.com CNAME   example.com

위 설정에서 sub.example.com은 example.com의 별칭 역할을 하며, 최종적으로는 192.168.1.1을 가리키게 됩니다.

3. 특징 및 장점

  • 유연한 관리: 서버의 IP가 바뀌더라도 A 레코드(원본) 하나만 수정하면 이를 가리키는 수많은 CNAME 별칭들은 자동으로 업데이트됩니다.
  • 서브도메인 관리: 여러 개의 서브도메인을 효율적으로 하나의 메인 도메인으로 연결할 때 유용합니다.

CNAME 레코드는 루트 도메인(예: example.com 자체)에는 가급적 사용하지 않는 것이 표준(RFC) 권장 사항입니다. 주로 www나 blog 같은 서브도메인 설정에 활용하시기 바랍니다.
