DDNS: 유동 IP 환경에서 고정 주소 사용하기

1. 유동 IP의 한계점

가정용 인터넷은 대부분 고정된 IP가 아닌, 접속할 때마다 주소가 바뀌는 유동 IP 방식을 사용합니다. 이 때문에 외부에서 내 집의 서버나 NAS에 접속하려고 할 때 매번 바뀐 IP를 확인해야 하는 불편함이 있습니다.

2. DDNS (Dynamic DNS)란?

DDNS는 호스트의 IP 주소가 변경될 때마다 DNS 레코드를 실시간으로 자동 갱신해 주는 서비스입니다. 덕분에 IP 주소가 아무리 바뀌어도 우리는 myhome.ddns.net과 같이 고정된 도메인 네임으로 내 서버에 접속할 수 있습니다.

3. 작동 원리 (Flow)

  1. 변경 감지: 공유기(Router)의 DDNS 클라이언트가 자신의 공인 IP가 바뀐 것을 감지합니다.
  2. 갱신 요청: 새로운 IP를 DDNS 서버로 전송하여 업데이트를 요청합니다.
  3. 레코드 업데이트: DDNS 서버가 도메인 주소에 연결된 IP를 새로운 주소로 갈아 끼웁니다.
  4. 전파 (Propagation): 업데이트된 정보가 전 세계 DNS 서버로 전파되어 어디서든 도메인을 통해 새로운 IP로 도달할 수 있게 됩니다.

DDNS는 “내 주소를 항상 최신 상태로 알려주는 자동 내비게이션”과 같습니다. 개인 서버 운영의 필수 요소입니다.
