DHCP: IP 자동 할당 프로토콜의 작동 원리

1. DHCP(Dynamic Host Configuration Protocol)란?

DHCP는 IP 네트워크에 접속하는 단말(호스트)들에게 IP 주소 및 기타 네트워크 설정 정보를 자동으로 할당하고 관리해 주는 프로토콜입니다.

2. 주요 기능과 장점

  • 자동 할당: 사용자가 일일이 서브넷 마스크, 게이트웨이, DNS 서버 정보를 입력할 필요 없이 서비스에 연결만 하면 자동으로 설정됩니다.
  • 중앙 집중 관리: 관리자는 DHCP 서버에서 IP 대역폭(Scope)과 임대 기간을 한곳에서 관리할 수 있어 효율적입니다.
  • 리소스 최적화: 한정된 IP 주소 자원을 필요한 기기에만 임대하고, 사용이 끝나면 회수하여 재사용하므로 IP 낭비를 방지합니다.

3. 임대(Lease) 기반 메커니즘

DHCP는 IP를 영구적으로 주는 것이 아니라 임대해 줍니다.

  • 갱신: 임대 기간이 만료되기 전에 클라이언트는 서버에 연장을 요청합니다.
  • 회수: 장치를 끄거나 네트워크를 이탈하여 갱신이 이루어지지 않으면, 해당 IP는 다시 풀(Pool)로 돌아가 다른 장치가 사용할 수 있게 됩니다.

DHCP는 복잡한 네트워크 환경에서 우리가 복잡한 설정 없이 와이파이를 즉시 쓸 수 있게 해주는 “자동 네트워크 가입 비서”와 같습니다.
