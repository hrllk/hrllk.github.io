도커(Docker) vs 가상머신(VM) 특징 비교

1. 가상머신 (Virtual Machine)

기존에는 물리 하드웨어를 효율적으로 나누어 쓰기 위해 VM 방식을 사용해 왔습니다.

  • 동작 방식: 호스트 OS 위에 하이퍼바이저를 띄우고, 그 위에 게스트 OS를 통째로 올리는 방식입니다. (OS 위에 OS)
  • 단점: OS 전체를 가상화하므로 매우 무겁습니다. 가상 OS 정보를 담은 이미지 파일 크기가 비대하며, 네이티브 환경이 아니기 때문에 실행 속도 저하가 발생합니다.

2. 도커 (Docker / Container)

도커는 VM의 단점을 극복하기 위해 등장한 컨테이너 기반 기술입니다.

  • 동작 방식: 하드웨어 가상화 대신 OS 수준의 격리를 수행합니다. 호스트 OS의 커널을 공유하며, 애플리케이션 실행에 꼭 필요한 바이너리와 라이브러리만 격리하여 담습니다.
  • 이점:
    1. 가벼움: 게스트 OS가 없으므로 이미지 크기가 작고 메모리 점유율이 낮습니다.
    2. 빠른 속도: 호스트의 자원을 거의 다이렉트로 사용하므로 실행 속도가 비약적으로 빠릅니다.
    3. 확장성: 빠르게 생성하고 파괴할 수 있어 클라우드 환경의 오토스케일링에 최적화되어 있습니다.

3. 어떤 기술을 선택해야 할까?

  • Docker: 마이크로서비스(MSA) 환경처럼 애플리케이션을 신속하게 배포하고 수평 확장(Scale-out)해야 할 때 최적입니다.
  • VM: 서로 다른 OS 환경에서의 동작을 완벽하게 격리하여 테스트해야 하거나, 아예 다른 운영체제 환경 자체가 필요할 때 적합합니다.

현대 백엔드 개발은 대부분 Docker를 중심으로 돌아갑니다. OS라는 무거운 짐을 벗어던지고 필요한 코드와 환경만 담아 어디서든 가볍게 실행할 수 있기 때문입니다.
